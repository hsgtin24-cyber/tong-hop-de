Bài 3: LÁT GẠCH

📌 Đề bài chi tiết

Quảng trường hình chữ nhật của thành phố ByteCity có kích thước $m \times n$. Quảng trường được lát bằng $m \times n$ viên gạch vuông kích thước $1 \times 1$. Sau một thời gian, những viên gạch cần được thay mới để đẹp hơn. Ban quản lý có $T$ viên gạch mới. Do không đủ số lượng gạch cần thiết nên họ quyết định chỉ thay mới những viên gạch nằm dọc theo đường biên của quảng trường, phần gạch còn lại ở trung tâm vẫn giữ nguyên như cũ. Lưu ý độ rộng của đường biên luôn bằng nhau ở các cạnh.

Yêu cầu: Hãy viết chương trình cho biết độ rộng tối đa của đường biên có thể thay mới được khi sử dụng không quá $T$ viên gạch.

Dữ liệu vào (LATGACH.INP): Một dòng ghi 3 số nguyên dương $m$, $n$ và $T$ ($3 \le m, n \le 10^9$; $1 \le T < m \times n$).

Kết quả ra (LATGACH.OUT): Một số nguyên là độ rộng tối đa cần tìm.

Ràng buộc:
  • 70% số điểm của bài: $3 \le m, n \le 10^6$.
  • 30% số điểm của bài: không ràng buộc gì thêm.

Ví dụ (Sample Test)

LATGACH.INP LATGACH.OUT Giải thích
5 7 33 2 Với quảng trường $5 \times 7$, dùng 32 viên gạch lát được đường biên có độ rộng bằng 2. (Lõi còn lại $1 \times 3$).
6 6 32 2 Với quảng trường $6 \times 6$, dùng 32 viên gạch lát được đường biên có độ rộng bằng 2. (Lõi còn lại $2 \times 2$).

🧠 Phân tích Toán học & Logic

Dạng bài: Toán học kết hợp Tìm kiếm nhị phân (Binary Search).

Gọi $x$ là độ rộng của đường biên ($x \ge 0$).

  • Diện tích toàn bộ quảng trường ban đầu là: $S_{total} = m \times n$.
  • Nếu lát biên có độ rộng $x$, phần trung tâm (lõi không lát) sẽ là một hình chữ nhật mới có:
    • Chiều dài lõi: $m' = m - 2x$
    • Chiều rộng lõi: $n' = n - 2x$
  • Diện tích phần lõi không lát là: $S_{core} = (m - 2x) \times (n - 2x)$.
  • Số lượng gạch cần dùng cho đường biên là:
    $$S(x) = (m \times n) - (m - 2x) \times (n - 2x)$$
  • Điều kiện: Phần lõi không thể có kích thước âm, và ít nhất phải $\ge 0$ (nếu lõi biến mất tức là đã lát toàn bộ). Do đó: $2x \le \min(m, n) \Rightarrow x \le \lfloor \frac{\min(m, n)}{2} \rfloor$.

Hàm $S(x)$ là một hàm đơn điệu tăng theo $x$. Khi độ rộng biên tăng, số gạch cần dùng chắc chắn tăng. Tính chất đơn điệu này là cơ sở tuyệt vời để áp dụng Tìm kiếm nhị phân.

CẢNH BÁO KIỂU DỮ LIỆU

Giới hạn $m, n \le 10^9 \Rightarrow m \times n \le 10^{18}$.

- Nếu code bằng C/C++: Bắt buộc phải sử dụng kiểu long long cho các biến $m, n, T, x$ và kết quả diện tích để tránh tràn số (Integer Overflow).
- Nếu code bằng Python: Python tự động xử lý số nguyên lớn (BigInt) nên không lo tràn số.

⚙️ Phân tích cách giải (Subtask)

Cách 1: Duyệt tuyến tính (Vét cạn) - 70% số điểm

Ý tưởng: Thử lần lượt các độ rộng $x$ từ $1$ tăng dần lên. Với mỗi $x$, tính số gạch cần dùng. Nếu số gạch $\le T$, ta ghi nhận kết quả và thử $x$ lớn hơn. Dừng lại khi số gạch vượt quá $T$ hoặc $x$ vượt quá giới hạn.

ans = 0
max_x = min(m, n) / 2
for x = 1 to max_x:
    gach_can_dung = (m * n) - (m - 2*x) * (n - 2*x)
    if gach_can_dung <= T:
        ans = x
    else:
        break
return ans

Nhược điểm: Độ phức tạp $O(\min(m, n))$. Khi $m, n = 10^9$, vòng lặp chạy đến $5 \times 10^8$ lần, gây quá thời gian (TLE) với subtask 30% cuối.

Cách 2: Tìm kiếm nhị phân (Tối ưu) - 100% số điểm

Recommended

Ý tưởng: Vì hàm số gạch là đơn điệu tăng theo $x$, ta áp dụng Tìm kiếm nhị phân trên không gian đáp án. Khoảng tìm kiếm ban đầu: low = 0, high = min(m, n) // 2. Thử mid, nếu thỏa mãn thì lưu đáp án và tìm ở nửa trên (tăng độ rộng), nếu không thì tìm ở nửa dưới.

low = 0, high = min(m, n) / 2, ans = 0
while low <= high:
    mid = (low + high) / 2
    gach_can = (m*n) - (m - 2*mid)*(n - 2*mid)
    if gach_can <= T:
        ans = mid // Lưu kỷ lục
        low = mid + 1 // Thử mở rộng biên
    else:
        high = mid - 1 // Thu hẹp biên 
import sys

# Tên file theo yêu cầu đề bài
INPUT_FILE = "LATGACH.INP"
OUTPUT_FILE = "LATGACH.OUT"

def check(x, m, n, T):
    if x <= 0: return True
    
    # Tính kích thước phần lõi bên trong
    core_m = m - 2 * x
    core_n = n - 2 * x
    
    # Đã lát toàn bộ quảng trường
    if core_m <= 0 or core_n <= 0:
        return (m * n) <= T
        
    so_gach_can = (m * n) - (core_m * core_n)
    return so_gach_can <= T

def solve(m, n, T):
    low = 0
    high = min(m, n) // 2
    ans = 0
    
    while low <= high:
        mid = (low + high) // 2
        if check(mid, m, n, T):
            ans = mid
            low = mid + 1
        else:
            high = mid - 1
            
    return ans

if __name__ == "__main__":
    # Đọc ghi file theo chuẩn thi HSG
    try:
        with open(INPUT_FILE, "r") as f:
            m, n, T = map(int, f.readline().split())
        res = solve(m, n, T)
        with open(OUTPUT_FILE, "w") as f:
            f.write(str(res))
    except Exception as e:
        pass
